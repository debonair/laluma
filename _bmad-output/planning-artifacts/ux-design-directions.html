Design decisions

  • Anonymous posts use warm gold card (`--color-anon-bg`) — distinct, celebrated, not shamed

Design decisions

  • Mode toggle is the top-most element in compose — impossible to miss or post in wrong mode
  • Anonymous identity pre-fetched on toggle (GET /anonymous-identity/:groupId) — revealed before posting
  • Post button copy changes per mode: "Post anonymously" vs "Post" — reinforces mode; no ambiguity
  • Preview card uses the same visual treatment as it will appear in feed — WYSIWYG
  • [Warm Adjective + Nature Noun] naming: "Quiet River 🌿" — warm, distinctive, non-identifying

Design decisions

  • Safety accent colour (plum) used for onboarding — this is where La Luma earns trust
  • Four concrete privacy promises in onboarding — not a vague "we care about you" statement
  • No "skip intro" on privacy screen — this is not optional; it's the trust ritual
  • Progress dots visible — she knows exactly where she is; no surprise length
  • Welcome screen is personal: her name; not a generic "you're all set"

Direction 5 — Upgrade Invite · Bonus, not wall
Upgrade sheet (bottom)
✨ La Luma Plus
There's more waiting for you
Unlock private groups, exclusive Luma Spaces, and advanced privacy controls — all designed with your safety in mind.
👥 Private groups with vetted members
✨ Access curated Luma Spaces events
🔒 Advanced privacy controls
🏡 👥 ✨ 👤

Design decisions

  • Upgrade surface is calm sky blue — deliberately distinct from safety plum; commercial ≠ safety signal
  • "There's more waiting for you" — invitation, not ultimatum; "Maybe later" is always visible
  • Features listed in safety-adjacent language: "vetted members", "privacy controls" — not just features
  • Bottom sheet — non-blocking; user can dismiss and continue; no hard paywall modal covering the whole screen
  • No countdown timer, no "X people upgraded today", no urgency whatsoever

Design decisions

  • Optimistic UI — reported content is visually dismissed immediately for the reporting user
  • Toast confirmation is warm and human: "Thank you for keeping this space safe" — not a case number
  • Toast uses success green, not safety plum — this is a resolved moment, not a safety warning
  • Moderation proceeds asynchronously; user sees outcome immediately and returns to the feed
  • Content warning design is consent-first: always offers "Show post" alongside "Report"

Design decisions

  • Lock screen always shows only "You have a new message in La Luma" — group name, content, anonymous status never revealed
  • This is design challenge #6 from discovery: notification privacy at OS level — handled as a first-class design decision
  • Even named-post notifications use the same format — consistency prevents inferring anonymous status from notification style
  • Default notification setting enforces this; users cannot opt into more verbose OS notifications

Design decisions

  • Error copy is always platform-owned: "This is on us, not you" — Experience Principle 5
  • Offline during post: draft saved automatically; her words are never lost
  • Warm amber (not red) for connectivity warning — not alarming; not an emergency
  • Same green leaf illustration as "caught up" empty state — calm continuity; not a system crash
  • "Recovery is always possible" (Principle 6): every error screen has a clear, calm next step

Design decisions

  • Anonymous post responses: "Someone responded to your post" + "Your original post was anonymous" — no identity revealed; no anonymous status inferred from notification format
  • Anonymous response thread uses the same warm gold card treatment — visual continuity from post to response
  • In-feed aha moment: name + first line of reply inline in the feed — warm card, not a modal; she stays in her community
  • Critical Success Moment #3: this is the moment she knows La Luma is real — designed as the most human moment in the app
  • Inbox groups responses by original post — all 3 responses to her post visible in one thread without losing context
